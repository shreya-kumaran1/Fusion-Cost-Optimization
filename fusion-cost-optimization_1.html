Efficient Testing
Avoid extra costs and save time on testing — aggregate tests into single queries and skip redundant tests with column-aware intelligence.
💰 ~2% warehouse savings
Why does combining 3 tests into 1 query save money?
✕
3 separate queries means your warehouse has to scan the same table 3 times, schedule 3 separate jobs, and you pay for 3 execution cycles — even though they're all reading the same data.
✓
1 combined query scans the table once, evaluates all 3 tests in a single pass, and returns all results together. One execution cycle, one table scan, one bill.
The math at scale: A project with 50 models averaging 3 tests each = 150 warehouse queries per run. With test aggregation, that drops to 50 queries — 100 fewer queries every single run.
✕ Before — 3 separate queries
SELECT count(*) FROM (SELECT ... test1 ...)
SELECT count(*) FROM (SELECT ... test2 ...)
SELECT count(*) FROM (SELECT ... test3 ...)
3 warehouse queries per model
3 table scans · 3 execution cycles · 3× the cost
✓ After — 1 combined query
WITH ... test1
WITH ... test2
WITH ... test3
SELECT ...
1 warehouse query per model
1 table scan · 1 execution cycle · 1× the cost
Column-aware testing goes even further
Fusion understands how columns flow through your DAG. If a column passes a test at one layer, and downstream models inherit that column without transforming it, the test doesn't need to run again — Fusion knows it would pass by definition.
♻
Example: customer_id passes a uniqueness test at the staging layer. The mart model inherits the same column unchanged — so Fusion skips the duplicate test entirely. Zero warehouse cost for that test.
